# Deploy Agent Suite

This guide covers using the Foundry deploy scripts shipped in `tagit-contracts` to deploy the three
ERC-8004 Agent infrastructure contracts:

| Contract | Script | Role |
|---|---|---|
| `TAGITAgentIdentity` | `DeployAgentIdentity.s.sol` | Soulbound ERC-721 agent registry |
| `TAGITAgentReputation` | `DeployAgentReputation.s.sol` | Agent feedback & scoring |
| `TAGITAgentValidation` | `DeployAgentValidation.s.sol` | Agent proof verification |
| All three | `DeployAgentSuite.s.sol` | Orchestrated single-broadcast deploy |

## Prerequisites

- [Foundry](https://book.getfoundry.sh/getting-started/installation) installed (`forge`, `cast`)
- A funded deployer wallet (EOA or Safe)
- An already-deployed `TAGITAccess` controller (see `.env.example`)
- RPC URL for the target network

---

## Environment Setup

Copy `.env.example` to `.env` and fill in the required values:

```bash
cp .env.example .env
```

```dotenv
# Required for all agent deploy scripts
PRIVATE_KEY=0x...
TAGIT_ACCESS=0x...          # Pre-deployed TAGITAccess controller

# Populated after Identity is deployed (required by Reputation & Validation standalone scripts)
AGENT_IDENTITY_ADDRESS=0x_DEPLOYED_AGENT_IDENTITY
AGENT_REPUTATION_ADDRESS=0x_DEPLOYED_AGENT_REPUTATION
AGENT_VALIDATION_ADDRESS=0x_DEPLOYED_AGENT_VALIDATION
```

<Callout type="warning">
Never commit a `.env` file with a live private key. Use a hardware wallet or a CI secrets manager.
</Callout>

## Option A â€” Single-Broadcast Suite Deploy (Recommended)

Deploy all three contracts in one transaction batch:

```bash
# Dry-run (no broadcast)
forge script script/deploy/DeployAgentSuite.s.sol \
  --rpc-url $OP_SEPOLIA_RPC_URL

# Live deploy + verify
forge script script/deploy/DeployAgentSuite.s.sol \
  --rpc-url $OP_SEPOLIA_RPC_URL \
  --broadcast \
  --verify
```

The suite script deploys in dependency order and wires registry addresses **inline** â€” no
pre-deployed addresses needed beyond `TAGIT_ACCESS`.

**Dependency order:**
1. `TAGITAgentIdentity` â€” no dependencies
2. `TAGITAgentReputation` â€” depends on Identity
3. `TAGITAgentValidation` â€” depends on Identity

At the end of the broadcast the console will emit `AGENT_*_ADDRESS` env var hints for
downstream scripts.

---

## Option B â€” Standalone Scripts (Step-by-Step)

Use the standalone scripts when deploying to multiple chains separately or when you need
fine-grained control.

### 1. Deploy Identity

```bash
forge script script/deploy/DeployAgentIdentity.s.sol \
  --rpc-url $OP_SEPOLIA_RPC_URL \
  --broadcast \
  --verify
```

Copy the printed `TAGITAgentIdentity` address into your `.env`:

```dotenv
AGENT_IDENTITY_ADDRESS=0x<address>
```

### 2. Deploy Reputation

```bash
forge script script/deploy/DeployAgentReputation.s.sol \
  --rpc-url $OP_SEPOLIA_RPC_URL \
  --broadcast \
  --verify
```

Requires `AGENT_IDENTITY_ADDRESS` in env. Automatically calls `setIdentityRegistry()` during
the broadcast.

### 3. Deploy Validation

```bash
forge script script/deploy/DeployAgentValidation.s.sol \
  --rpc-url $OP_SEPOLIA_RPC_URL \
  --broadcast \
  --verify
```

Same requirements as Reputation. Calls `setIdentityRegistry()` inline.

## Post-Deploy Checklist

After any deploy path, complete the following:

- [ ] Verify all three contracts on the block explorer
- [ ] Call `setAccessController()` on each contract if `TAGIT_ACCESS` was not set at deploy time
- [ ] Run `RegisterSageAgent.s.sol` to register Agent #1
- [ ] Grant `AGENT_VALIDATOR` capability to validator addresses via BIDGES
- [ ] Transfer contract ownership to the multisig
